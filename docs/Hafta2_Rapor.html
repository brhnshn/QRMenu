Token Akışı

1. Müşteri QR kodu okur → /qr/{masaNo}
2. QRController: Token = CSPRNG ile 256-bit rastgele üretir
3. TokenHash = SHA256(Token) → Oturum tablosuna kaydedilir
4. Token → HttpOnly + Secure cookie ile tarayıcıya yazılır
5. Middleware — her istekte cookie okur → DB'den oturum doğrular
6. SonIslemTarihi güncellenir (Sliding Expiration)
7. 90 dk süre aşımında oturum kapanır

5. Düzeltilen Hatalar

Hata Çözüm
Cookie Secure flag'i localhost'ta çalışmıyor Development ortamında Secure = false yapıldı
CreateSessionAsync mevcut TokenHash'i sıfırlıyordu Yeni oturum oluştururken eski oturumun hash'i korunacak şekilde düzeltildi
Masa değiştirmede oturum karışıyordu Cookie silinip yeni token oluşturularak düzeltildi
QrKodViewModel namespace çakışması Controllers'dan ViewModels klasörüne taşındı
