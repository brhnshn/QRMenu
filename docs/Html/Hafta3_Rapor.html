3. Sepet İş Kuralları (Business Logic)

Duplikat Ürün Mantığı

Aynı müşteri aynı ürünü (Örn: Çay) arka arkaya 2 defa eklerse veritabanına iki ayrı satır açılmaz. Sadece eski satır bulunur, adet +1 yapılır.

// Aynı ürün + aynı opsiyonlarla sepette var mı?
var mevcutDetay = await _context.SepetDetaylar
    .FirstOrDefaultAsync(sd => sd.SepetId == sepetId
                            && sd.UrunId == urunId
                            && sd.SeciliOpsiyonlar == opsiyonJson);

if (mevcutDetay != null) {
    mevcutDetay.Adet += adet;
} else {
    // Yeni SepetDetay nesnesi yarat ve kaydet
}

4. Düzeltilen Kritik Hatalar

Hata Kaynağı Açıklama Çözüm & Fix
QRMenuDbContext (SQL DbSet) Opsiyonlar tablosu projede yanlışlıkla 2 kez tanımlanmıştı (Satır 14 ve 19) bu da Migration'ların hata fırlatmasına sebep olacaktı. Grep_search ile arandı, test edildi ve 19. satırdaki DbSet<Opsiyon> Opsiyonlar duplikesi kalıcı silindi.
Birleştirilmiş Test Json Hatası Assert.Contains("Büyük Boy") opsiyon testi, Türkçe karakterlerin JSON'da B\u00FCy\u00FCk Boy diye Unicode kaçması yüzünden Fail (Hatalı) döndürdü. Metin okutmak yerine veritabanındaki PrimaryKey Id si okutuldu: Assert.Contains("\"Id\":100"). Test %100 başarıya ulaştı.
Supabase Deployment Logu Geliştirme ağından Supabase 5432 session portuna bağlanılamadı (Timeout Error). Geliştirici konsolunda bu Log dosyaları tespit edildi. Bağlantı stringindeki port geçici olarak 6543 ile bypass edildi ve sistemin duraksaması engellendi. (Transaction Pooler Mimarisi)
