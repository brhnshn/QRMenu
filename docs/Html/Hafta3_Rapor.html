2. Sistem Mimarisine Eklenen Dosyalar

Bu hafta Sepet sistemini üç ana katmanda (Core, Data, Web) hayata geçirdik.

  • Core: ISepetService.cs — Uygulamanın sepet iş kurallarını tanımlayan Interface sınıfı.
  • Data: SepetService.cs — QRMenuDbContext (SQL bağlantısı) kullanarak Sepet entity'sini yöneten servis sınıfı. Mükerrer ürün eşleştirmesi yapıyor.
  • Web/Controllers: SepetController.cs — Frontend ile Backend arasındaki köprü. Müşterinin tıkladığı butonları Asenkron (AJAX) isteklerle alır.
  • Web/Views:
    • Sepet/Index.cshtml: Müşteri sepet sayfasına bakarken listeyi, toplamı, arttır/azalt düğmelerini üretir.
    • Menu/Index.cshtml: Menü ürün kartlarına "Sepete Ekle" butonu eklendi, opsiyon modal'ı (açılır pencere) JS scriptleri yazıldı. Kayan sepet ikonu eklendi.
  • Web/css: sepet.css — Sepet sayfası (Modern Card Layout) Responsive (mobil odaklı) ekran tasarımları yapıldı.
  • Tests: SepetServiceTests.cs — Servisin doğru çalışması için test kuralları barındırır.

3. Güvenlik, Hata Ayıklama (Log) ve Bağlantı Çözümleri

Token-Based Sepet Sistemi: Redis yerine, Veritabanı seçilmesinin en büyük kârı müşteri yetkilendirmesiydi. Gelen her Controller isteğinde (`/sepet`, `/sepet/ekle` vs) ilk önce GetOturumIdAsync() fonksiyonuyla şifreli cookie tokenı DB'deki SHA256 Hashiyle kontrol ediliyor. Token yoksa veya bozuksa anında Unauthorized (401) döndürülüyor.
Çözülen Kritik Bug 1: QRMenuDbContext içerisinde Opsiyonlar tablosu aynı anda 2 kere (14. Satır ve 19. satır) DbSet olarak tanımlandığı tespit edildi. Migration işlemini kırmaya çalışan bu hata, tespit edilerek silindi.
Çözülen Kritik Bug 2: Okul ağı ve dış ağ bağlantılarından dolayı PostgreSQL (Supabase) port 5432 Timeout hatası veriyordu. appsettings.json dosyasında `Port=6543;` Transaction Pooler olarak değiştirildi/denendi, ardından mobil veriye geçilerek 5432 bağlantı limiti güvenli aşıldı ve veritabanı başarıyla senkronize edildi.
Çözülen Kritik Bug 3 (Test Hatası): JSON'a dönüştürülen TürkçeOpsiyon karakterleri Büyük Boy -> B\u00FCy\u00FCk Boy testte hata fırlatıyordu. String aramak yerine test dosyası güncellenip doğrudan Id numarası `Assert.Contains("\"Id\":100")` ile düzeltildi ve tüm (23/23) testler başarıyla PASSED (Yeşil) yandı.

4. Yazılım Tasarımı ve Veritabanı Mantığı

Eğer müşteri peş peşe 2 kere "Türk Kahvesi" eklerse, veritabanına 2 ayrı satır olarak kaydetmek gereksiz alan israfıydı. Yaptığımız Duplikat Optimizasyonu şudur:

// Aynı ürün + aynı opsiyonlarla sepette var mı?
var mevcutDetay = await _context.SepetDetaylar
    .FirstOrDefaultAsync(sd => sd.SepetId == sepetId
                            && sd.UrunId == urunId
                            && sd.SeciliOpsiyonlar == opsiyonJson);

if (mevcutDetay != null) {
    // Varsa 2 ayrı satır ekleme, sadece eski satırın "Adet" kolonunu +1 yap
    mevcutDetay.Adet += adet;
} else {
    // Yoksa Yeni Satır INSERT işlemi yap
}
